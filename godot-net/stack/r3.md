# R3

> **Scope**: Reactive programming with R3 (Cysharp) in Godot 4 .NET — Observable/Observer patterns, ReactiveProperty, Subject types, subscription disposal, Godot-specific frame providers, signal integration, async operators, and testing with FakeTimeProvider/FakeFrameProvider.
> **Load when**: authoring reactive streams with R3, using ReactiveProperty or Subject, managing subscription lifetimes, wiring Godot node events as observables, using frame-based operators, integrating async/await with reactive pipelines, testing time-dependent or frame-dependent observables, choosing between R3 and Chickensoft.Sync for reactive programming.

## Core Concepts

R3 replaces `IObservable<T>` / `IObserver<T>` interfaces with **abstract classes** to enable strict lifecycle control, subscription tracking, and resumable error handling.

```csharp
// Base types
public abstract class Observable<T>
{
    public IDisposable Subscribe(Observer<T> observer);
}

public abstract class Observer<T> : IDisposable
{
    public void OnNext(T value);
    public void OnErrorResume(Exception error);   // does NOT terminate subscription
    public void OnCompleted(Result result);       // Result.Success or Result.Failure(ex)
}
```

Key differences from traditional Rx:

| Aspect | Traditional Rx | R3 |
|--------|---------------|----|
| Core types | `IObservable<T>` / `IObserver<T>` interfaces | Abstract classes |
| Error handling | `OnError` terminates subscription | `OnErrorResume` continues by default |
| Completion | Separate `OnCompleted` / `OnError` | Single `OnCompleted(Result)` |
| Scheduling | `IScheduler` (slow, allocating) | `TimeProvider` + `FrameProvider` |
| Subscription return | Separate `IDisposable` | Observer itself (zero extra allocation) |

**Hot vs Cold:** Cold observables produce values per subscriber (e.g., `Observable.Range`). Hot observables emit regardless of subscriptions (e.g., `Subject<T>`, `ReactiveProperty<T>`).

## Subject Types

| Type | Value Stored | Duplicates | Use Case |
|------|-------------|------------|----------|
| `Subject<T>` | None | All published | Event bus, fire-and-forget |
| `BehaviorSubject<T>` | Latest value | All published | Cached state |
| `ReactiveProperty<T>` | Latest value | Suppressed by default | Observable model property, UI binding |
| `ReplaySubject<T>` | N values or time window | All buffered | Late-subscriber replay |
| `ReplayFrameSubject<T>` | N frames | All buffered | Frame-windowed replay |

All Subject types call `OnCompleted()` on disposal by default (ensuring downstream subscriptions terminate). Pass `false` to `Dispose(false)` to skip this.

## ReactiveProperty

`ReactiveProperty<T>` is a BehaviorSubject with duplicate elimination and a `.Value` setter — the primary tool for observable model properties.

```csharp
public class EnemyModel : IDisposable
{
    // Writable internally, exposed as read-only outside
    private readonly ReactiveProperty<long> _currentHp;
    private readonly ReactiveProperty<bool> _isDead;

    public ReadOnlyReactiveProperty<long> CurrentHp => _currentHp;
    public ReadOnlyReactiveProperty<bool> IsDead => _isDead;

    public EnemyModel(long initialHp)
    {
        _currentHp = new ReactiveProperty<long>(initialHp);
        // Derive IsDead automatically from CurrentHp
        _isDead = _currentHp.Select(hp => hp <= 0).ToReactiveProperty();
    }

    public void TakeDamage(long amount) => _currentHp.Value -= amount;

    public void Dispose()
    {
        _currentHp.Dispose();
        _isDead.Dispose();
    }
}

// Consumer (Godot node)
public partial class EnemyHUD : CanvasLayer
{
    private IDisposable _subscription;

    public void Init(EnemyModel model)
    {
        var builder = Disposable.CreateBuilder();
        model.CurrentHp.Subscribe(hp => _hpLabel.Text = $"HP: {hp}").AddTo(ref builder);
        model.IsDead.Where(dead => dead).Subscribe(_ => Hide()).AddTo(ref builder);
        _subscription = builder.Build();
    }

    public override void _ExitTree() => _subscription?.Dispose();
}
```

**Force notification** (even if value unchanged):
```csharp
property.OnNext(property.Value); // bypasses duplicate suppression
```

**Custom value validation** via override:
```csharp
public sealed class ClampedReactiveProperty<T>(T initialValue, T min, T max)
    : ReactiveProperty<T>(initialValue) where T : IComparable<T>
{
    protected override void OnValueChanging(ref T value)
    {
        if (Comparer<T>.Default.Compare(value, min) < 0) value = min;
        else if (Comparer<T>.Default.Compare(value, max) > 0) value = max;
    }
}
```

## Subscription Disposal

Always dispose subscriptions. Choose the right utility for each scenario:

| Utility | Allocation | Thread-safe | When to use |
|---------|-----------|-------------|-------------|
| `Disposable.Combine(d1…d8)` | None | No | Fixed, known count (≤8) — best performance |
| `Disposable.CreateBuilder()` | None (struct) | No | Multiple subscriptions created at setup |
| `DisposableBag` (struct field) | Minimal | No | Dynamic subscriptions added over lifetime |
| `CompositeDisposable` | Class | Yes | Dynamic + multi-threaded scenarios |

```csharp
// Static — best for known subscription counts
IDisposable d = Disposable.Combine(sub1, sub2, sub3);

// CreateBuilder — zero-alloc construction phase
var b = Disposable.CreateBuilder();
obs1.Subscribe(OnA).AddTo(ref b);
obs2.Subscribe(OnB).AddTo(ref b);
IDisposable d = b.Build(); // call once; builder is consumed

// DisposableBag — dynamic, stored as field
private DisposableBag _bag;
obs.Subscribe(OnX).AddTo(ref _bag);
// dispose all:
_bag.Dispose();

// CompositeDisposable — thread-safe
private readonly CompositeDisposable _cd = new();
obs.Subscribe(OnX).AddTo(_cd);
_cd.Dispose();
```

**Godot shortcut:** `AddTo(Node node)` auto-disposes the subscription when the node exits the scene tree:
```csharp
Observable.EveryUpdate()
    .Subscribe(_ => GD.Print("tick"))
    .AddTo(this); // disposed automatically on _ExitTree
```

## Godot Node Integration

Subscribe in `_Ready` (or after DI resolves), dispose in `_ExitTree`. Use `AddTo(this)` to avoid manual disposal bookkeeping.

```csharp
public partial class Player : CharacterBody2D
{
    private IDisposable _subscriptions;

    public override void _Ready()
    {
        var builder = Disposable.CreateBuilder();

        // Frame-based poll
        Observable.EveryUpdate(GodotFrameProvider.Process)
            .Subscribe(_ => HandleMovement())
            .AddTo(ref builder);

        // Button as observable
        GetNode<Button>("AttackButton")
            .OnPressedAsObservable()
            .ThrottleFirstFrame(10, GodotFrameProvider.Process)
            .Subscribe(_ => Attack())
            .AddTo(ref builder);

        _subscriptions = builder.Build();
    }

    public override void _ExitTree()
    {
        _subscriptions?.Dispose();
    }
}
```

### Godot Providers

```csharp
// Time providers (wall-clock, async)
GodotTimeProvider.Process         // _Process delta
GodotTimeProvider.PhysicsProcess  // _PhysicsProcess delta

// Frame providers (game loop, synchronous)
GodotFrameProvider.Process        // each _Process tick
GodotFrameProvider.PhysicsProcess // each _PhysicsProcess tick
```

When no provider is specified, operators use `ObservableSystem.DefaultFrameProvider` / `DefaultTimeProvider` — set by the plugin autoload to `GodotFrameProvider.Process` / `GodotTimeProvider.Process`.

### UI Extensions

```csharp
button.OnPressedAsObservable()                    // BaseButton → Observable<Unit>
slider.OnValueChangedAsObservable()               // Range → Observable<float>
lineEdit.OnTextChangedAsObservable()              // LineEdit → Observable<string>
textEdit.OnTextChangedAsObservable()              // TextEdit → Observable<Unit>

label.SubscribeToText(model.Name);                // Observable<string> → Label.Text
```

### Signal as Observable

```csharp
// Convert any Godot signal to Observable
node.SignalAsObservable<string>("my_signal")
    .Subscribe(data => HandleSignal(data));

// CancellationToken that fires when a signal emits
using var tokenDisposable = node.CancelOnSignal("destroyed");
Observable.EveryUpdate(cancellationToken: tokenDisposable.Token)
    .Subscribe(_ => GD.Print("alive"));
```

## Frame-Based Operators

Prefer frame-based operators over time-based for game logic — they follow the game loop exactly.

```csharp
// Every frame
Observable.EveryUpdate()
Observable.EveryUpdate(GodotFrameProvider.PhysicsProcess)

// Throttle: emit first value in window of N frames, ignore rest
source.ThrottleFirstFrame(10)

// Throttle: emit last value at end of N-frame window
source.ThrottleLastFrame(10)

// Debounce: emit only after N frames of silence
source.DebounceFrame(5)

// Skip first N frames
source.DelayFrame(3)

// Emit every N frames
Observable.IntervalFrame(60)

// Poll for value changes each frame
Observable.EveryValueChanged(this, x => x.Position)
    .Subscribe(pos => GD.Print(pos));

// One-shot: wait until next frame then complete
Observable.NextFrame(GodotFrameProvider.Process)
```

## Key Operators

```csharp
// --- Filtering ---
source.Where(x => x > 0)
source.Take(5)                    // first 5 values
source.Skip(3)                    // skip first 3
source.TakeUntil(other)           // until other emits
source.TakeUntilCanceled(ct)      // until CancellationToken

// --- Transformation ---
source.Select(x => x * 2)
source.SelectMany(x => GetChildren(x))
source.Distinct()
source.DistinctUntilChanged()

// --- Combining ---
Observable.Merge(s1, s2)          // any of the sources
Observable.CombineLatest(s1, s2, (a, b) => (a, b))  // latest from both
Observable.Zip(s1, s2, (a, b) => (a, b))            // paired by index
source.WithLatestFrom(other, (a, b) => (a, b))      // source ticks, other provides latest

// --- Time / Frame ---
source.Debounce(TimeSpan.FromMilliseconds(300))
source.Throttle(TimeSpan.FromSeconds(1))
source.Delay(TimeSpan.FromSeconds(0.5))

// --- Collection ---
source.ToLiveList()               // captures all emitted values (for testing / binding)
source.Chunk(3)                   // buffer into arrays of 3
```

## Async Integration

Use `SelectAwait` / `WhereAwait` to integrate async operations into observable pipelines. Control concurrency with `AwaitOperation`:

| Strategy | Behavior | Game use case |
|----------|----------|---------------|
| `Sequential` | Queue, process one at a time | Ordered requests |
| `Drop` | Ignore new while in-flight | Prevent double-tap (most common in games) |
| `Switch` | Cancel previous, start new | Search-as-you-type, dynamic target lock |
| `Parallel` | All concurrent | Independent fetches |
| `SequentialParallel` | Parallel, ordered output | Parallel + deterministic results |
| `ThrottleFirstLast` | First + last during execution | Debounce with feedback |

```csharp
// Prevent duplicate attack while previous is resolving
attackButton.OnPressedAsObservable()
    .SelectAwait(async (_, ct) =>
    {
        return await PerformAttackAsync(ct);
    }, AwaitOperation.Drop)
    .Subscribe(result => ApplyResult(result));

// Cancel ongoing search when new input arrives
searchInput.OnTextChangedAsObservable()
    .Debounce(TimeSpan.FromMilliseconds(300))
    .SelectAwait(async (text, ct) => await SearchAsync(text, ct), AwaitOperation.Switch)
    .Subscribe(results => DisplayResults(results));
```

## Error Handling

R3's default: **errors do not terminate the subscription**. `OnErrorResume` is called instead of `OnError`.

```csharp
// Default behavior: stream continues after error
source.Subscribe(
    onNext: x => Process(x),
    onErrorResume: ex => GD.PrintErr($"Non-fatal: {ex}"),
    onCompleted: result => { if (result.IsFailure) HandleFatal(result.Exception); }
);

// Opt into traditional termination behavior
source.OnErrorResumeAsFailure()
    .Subscribe(x => Process(x));

// Global unhandled exception handler (set once at startup)
ObservableSystem.UnhandledExceptionHandler = ex => GD.PrintErr($"Unhandled R3: {ex}");
```

## Testing

Use `FakeTimeProvider` and `FakeFrameProvider` for deterministic tests. Collect emitted values with `ToLiveList()`.

```csharp
using Microsoft.Extensions.Time.Testing;

// Time-based test
var fakeTime = new FakeTimeProvider();
var list = Observable.Timer(TimeSpan.FromSeconds(5), fakeTime).ToLiveList();

fakeTime.Advance(TimeSpan.FromSeconds(4));
list.AssertIsNotCompleted();

fakeTime.Advance(TimeSpan.FromSeconds(1));
list.AssertIsCompleted();
list.AssertEqual([Unit.Default]);

// Frame-based test
var cts = new CancellationTokenSource();
var frames = new FakeFrameProvider();
var frameList = Observable.EveryUpdate(frames, cts.Token)
    .Select(_ => frames.GetFrameCount())
    .ToLiveList();

frames.Advance(3);
frameList.AssertEqual([0, 1, 2]);

cts.Cancel();
frameList.AssertIsCompleted();
```

## ObservableTracker (Development Only)

Enable during development to detect subscription leaks:

```csharp
ObservableTracker.EnableTracking = true;
ObservableTracker.EnableStackTrace = true; // captures allocation site

// Enumerate all active subscriptions
ObservableTracker.ForEachActiveTask(state =>
    GD.Print($"{state.FormattedType} | added: {state.AddTime}"));
```

**Disable in production** — `EnableTracking` and `EnableStackTrace` add overhead. Access the tracker window in Godot via **Debugger → ObservableTracker**.

## Performance

- R3 vs dotnet/reactive: `Observable.Range(1, 10000).Subscribe()` ~100 ns / 0 allocs in R3 vs ~7000 ns / multiple allocs in dotnet/reactive.
- Prefer `Disposable.Combine` / `CreateBuilder` over `CompositeDisposable` when subscriptions are created synchronously at setup (no thread safety needed, zero allocation).
- Prefer `DisposableBag` as a struct field over `CompositeDisposable` for dynamically-added subscriptions (avoids class allocation).
- Use frame-based operators (`ThrottleLastFrame`, `DebounceFrame`) for game logic — they do not allocate timers.
- `ObservableTracker` adds per-subscription overhead — enable only with `#if DEBUG` guards.

**When to choose R3 vs Chickensoft.Sync:**
- **R3** — when you need LINQ-style operator composition, async integration (`SelectAwait`), high throughput (signal processing, input streams), or cross-platform compatibility.
- **Chickensoft.Sync** — when you need deterministic reentrancy-safe reactive state for game model ↔ UI binding with the Chickensoft ecosystem.

## Anti-patterns

- **Not disposing subscriptions** — every `Subscribe` without `AddTo` or manual `Dispose` is a leak; use `AddTo(this)`, `DisposableBag`, or `CompositeDisposable`.
- **Using IScheduler** — R3 has no `IScheduler`; use `TimeProvider` and `FrameProvider` parameters instead.
- **Subscribing in `_Process`** — creates a new subscription every frame; subscribe once in `_Ready`.
- **Using time-based operators for game logic** — `Debounce(TimeSpan)` allocates a timer; prefer `DebounceFrame(n)`.
- **Exposing `ReactiveProperty<T>` directly** — always expose `ReadOnlyReactiveProperty<T>` to consumers; keep the mutable field private.
- **Expecting traditional `OnError` termination** — R3 calls `OnErrorResume` by default (stream continues); add `OnErrorResumeAsFailure()` if termination is required.
- **Using `CompositeDisposable` for single-threaded subscriptions** — prefer `DisposableBag` (struct) or `Disposable.Combine` to avoid unnecessary heap allocation.
- **Forgetting to call `Dispose()` on Subjects / ReactiveProperties** — Subject disposal calls `OnCompleted()` on all subscribers; omitting it leaves dangling subscriptions.
- **Chaining `Dispose(false)` without understanding consequences** — skips the downstream `OnCompleted` notification; only use when you intentionally want to silently terminate.
