# UniKit Rules

Community-maintained knowledge base rules for [UniKit AI](https://github.com/NintendaDev/unikit-ai) game development toolkit.

## Structure

```
<engine>/
  core/         Core rules (always loaded): code style, design principles, testing, performance
  stack/        Stack rules (on demand): framework-specific patterns and conventions
    references/ Supplementary reference docs for complex frameworks
```

### Supported Engines

| Engine | Core | Stack |
|--------|------|-------|
| Unity | 5 rules | 8 rules |
| Godot 4 | 5 rules | - |
| Godot 4 .NET | 5 rules | - |
| Unreal Engine 5 | 6 rules | 15 rules |

## How It Works

UniKit CLI fetches rules from this repository via `raw.githubusercontent.com`. The `manifest.json` file is the entry point - it lists all available rules with their metadata.

```bash
# Install a rule from this registry
unikit-ai rules install dotween

# List all available rules
unikit-ai rules list

# Use a custom/forked registry
unikit-ai rules registry https://raw.githubusercontent.com/your-org/your-rules/main
```

## Rule File Format

Each rule file has YAML frontmatter and a standard header:

```markdown
---
version: 1.0.0
---

# Rule Name

> **Scope**: What this rule covers
> **Load when**: Keywords and contexts that trigger loading

---

## Section 1

Rules, code examples, patterns...
```

### Required Fields

- **Frontmatter `version`**: Semver version. Must be bumped when content changes.
- **`> **Scope**:`**: Brief description of what the rule covers.
- **`> **Load when**:`**: Comma-separated keywords/contexts for dynamic loading.

### Optional

- **`> **References**:`**: Links to reference files in `references/` subdirectory.
